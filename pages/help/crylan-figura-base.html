help and guides: crylan minecraft figura base

guide to using the base and updating your avatar to new versions of the base

the information on this page is correct, but its format and wording is subject to change.
last updated: 2024/12/26

product requirements [#]

This avatar base requires the following software:

  • Minecraft (Java Edition)
  • Figura and a compatible Minecraft mod loader
  • Blockbench, downloaded or using the web app.

customizing the base [#]

Let's get you set up to recolor & paint the textures in Blockbench!

  1. Unpack the *.zip folder you downloaded to a new folder. Name that folder whatever you like - if your skin is based on a character, I'd reccomend naming it after the character.

  2. Open Blockbench. You will want to edit the textures in Blockbench to make sure your skin will display correctly in the rewrite versions of figura.

  3. Select "Open Model", and navigate to your unpacked folder. Select the player_model.bbmodel

  4. Use "Paint" mode to recolor the textures as you please! (Tip: Setting the Paint Bucket tool to Colors will recolor everything on the texture of a matching color, while Connected Colors will work like a normal fill bucket.)

  5. Save your work (regularly) by saving over the player_model.bbmodel, and saving the texture over the texture.png if it isn't saving over it automatically.

  6. Once all your work is saved, move it into the figura avatars folder and you should see it appear in-game!

  7. Go ahead and update the avatar.json file to give yourself some credit!

  8. Upload the avatar in-game!

If you are having issues with the script, it is likely that you are using the wrong version of the base or Figura. Check the below guide for help with that.

finding the avatars folder [#]

  1. Open Minecraft with Figura installed.

  2. Open Options, and click the Figura icon.

  3. There should be a file folder button to open your avatars folder. Click it!

changing the avatar details [#]

  1. Open "avatar.json" with your text editor of choice (notepad will work just fine).

  2. In "authors", replace "your_username_here" with your username!

  3. Change the display name of the avatar by changing the value for "name" ("Crylan Base" by default).

updating older avatars for new versions of figura [#]

Updated Figura and now it can't find your avatar, the texture is wrong, or the script is crashing?

This base only officially supports versions listed on the ko-fi store page. Other versions may display the avatar, but will likely fail to load the script.

figura isn't showing my avatar in the wardrobe [#]

Open the avatars folder. (how?)

  • My avatar is in that folder: Skip to generic updating steps.
  • My avatar is NOT there: Proceed to the next section.

moving your avatar to the new folder [#]

At some point, Figura moved its folders around. Your avatar is likely in another folder!

  1. Open the figura folder in a new window.
    Windows: Press Windows+R, type in %appdata%/.minecraft/figura and click OK.

  2. If there is a folder named model_files, your old avatar should be in there.

  3. Move your old avatar into the avatars folder. (where?)

generic updating steps [#]

  1. Back up the old avatar by copying the folder to another location, like your desktop.

  2. Download the appropriate version of the avatar base for your Figura version, and unpack the *.zip file to a new folder.

  3. If your old avatar is packed into a *.zip folder, unpack it.

  4. Copy only the following files from the new base to your avatar's folder: "script.lua", "avatar.json". These new files will replace their old versions, if any exist.

  5. Update the contents of avatar.json how?

custom texture isn't displaying [#]

This issue usually only happens if you edit the texture with anything other than Blockbench, or if you replace the *.bbmodel file in your avatar folder.

  1. Open your model in Blockbench. (Instructions on this are under using the base!)

  2. You should notice that the texture your model is using is not the one you customized.
    Go ahead and right click that texture and select "Change File", then select the correct texture file.
    (You'll be able to see it on the model, so it should be clear if you've selected the wrong texture.)

  3. Save over your previous player_model.bbmodel with this fixed one, then check in-game to make sure it's working correctly.

And now you should be good to go!

script is broken [#]

Using this base with an incompatible version of Figura will likely cause part or all of the script to break.
This isn't the end of the world, but it might look rather unsightly.
You can contact me to request an update to the base, or you can try to edit the script yourself if you're brave!

Still having trouble? Feel free to contact me! I'll do my best to help you out :)
